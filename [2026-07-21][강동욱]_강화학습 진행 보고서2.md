### v2 변경사항2(학습 방식)
- reward에 추가된 측면, 후면 타격 보상은 tank가 스스로 깨닫기 어려운 보상임(기본적으로는 서로 마주보고만 전투를 하기 때문)
- 따라서 상대 tank를 측면에서 혹은 후방에서 습격한다는 시나리오를 만들어 학습에 추가
	- env 128개 중 60%는 기존 방식대로 20%는 측면 20%는 후면 시나리오로 할당하는 식으로 진행
	- 승률 평가는 당연히 기존대로 서로 정면을 바라본 채 공평하게 진행

## v3 model
- 특정 진영에선 승률이 매우 높은 반면 조금만 변형을 줘도 동일 상대에게 승률이 확 떨어지는 문제가 v2 이후 학습에서 존재했었음(폐기된 이전 v3)
	- v2를 상대로 (A 42.5% / B 85%) 의 승률을 보였음
- 이때부터 발생했던 과적합 해결을 위해 학습을 보다 일반화시켜 진행(처음부터 이렇게 했다면 수렴이 안 됐을 것이라 판단하였음)
### v3 변경 사항
1. 각 tank 시작 위치를 ±3m 랜덤 이동 + 방향 ±10° 랜덤으로 둠
2.  128개 env 중 절반은 A진영 절반은 B진영에서 학습
3. 이전엔 v1은 룰봇만 v2는 v1만 상대했다면 v3는 로테이션으로 여러 상대와 전투 
	1. 로테이션 풀은 직전 3개 세대를 대상으로 짜여지게끔 설계(v7이면 v4~6을 상대)

# ES(Evaluation Strategies) 방식 정리(유전 알고리즘)
- 현 모델의 head 파라미터는 2827개
- 각 파라미터에 변형을 주어 16개의 서로 다른 모델을 생성
```
θ+σ·ε₁, ..., θ+σ·ε₁₆
```
- 여기서 세타는 2827차원의 파라미터 벡터
- 시그마는 0.02(노이즈 상수)
- 앱실론은 표준 정규분포를 따르는 16개의 2827차원 벡터
- 즉 head 파라미터에 가우시안 분포를 따르는 variation을 준 것
- 이 16개 모델을 16개 각 환경에서 전투를 치르게 하고 16개 모델을 [- 0.5, +0.5] 범위로 정규화(reward가 가장 높은 모델은 +0.5 가장 낮은 모델은 -0.5에 해당되도록 정규화)
- 이 가중치들을 이용해 파라미터 변화량에 대한 가중 평균을 계산(reward가 높은 쪽으로 파라미터가 이동할 것임)
- 계산된 방향에 대해 학습률을 곱해서 이동(lr = 0.01)

- 긍정적으로 보이는 점은 이동하는 양상이 PPO와는 조금 다르다는 점에 있음
- 아직 장기간 학습을 시켜둔건 아니기에(PPO 위주로 학습을 했었음) ES 방식으로도 추가적으로 학습을 시켜볼 예정
