# 3v3 탱크 전투 강화학습 파이프라인

## 전체 흐름

```text
1) 기준 맵(map1)을 jitter → 총 10개 map 확보

2) Rule bot 제작
   → 3 vs 3 랜덤 파라미터 모델 대결
   → 육안 검증

3) 위 rule bot을 지도학습(BC)으로 흉내
   → policy가 어느 정도 똑똑해진 상태에서 시작

4) 강화학습 phase 1
   고정 rule bot 3 vs 학습 policy 3
   → 한쪽만 gradient 업데이트

   ↓

   전투 구도가 학습 policy 쪽으로 기울면 해당 policy를 v1로 고정

   ↓

   phase 2
   v1 vs 새 학습 policy
   → v2 → v3 → ...

   ↓

5) 매 전투 random map + n_envs 500 ~ 1000 병렬 학습
```

## 1. 후진 이동 방식

backward.mp4

Tank의 최초 heading 방향과 목표 경로 방향의 각도 차이가 15도 이상 나면, 바로 회전하지 않고 후진 후 전진하는 방식으로 이동한다.

이 방식은 목표 방향이 현재 차체 방향과 크게 어긋나 있을 때, 제자리 회전이나 비효율적인 큰 회전 경로를 줄이기 위해 사용한다.

## 3. Rule bot 설계 및 검증

Rule bot은 강화학습 이전에 기본적인 전투 행동을 정의하기 위해 제작하였다.

핵심 아이디어는 다음과 같다.

* 각 tank는 역할별로 지정된 post로 이동한다.
* post에 도착하면 적을 관측하며 대기한다.
* 일정 시간 동안 적이 보이지 않으면 다음 post로 이동한다.
* HP가 낮아지면 spawn으로 후퇴한다.
* visible 상태이고 LOS가 clear한 적을 조준한다.
* reload가 완료되면 발사한다.

### Rule bot 이동 로직

```python
if 죽음:
    move_dest = None

elif HP < 30:
    move_dest = own_spawn                    # 후퇴

elif 현재 post까지 거리 < 6m:                 # 도착 판정
    if 시야에 적 없음 and 마지막 관측 후 5초+:
        post_idx += 1                        # 순회, 다음 post
        move_dest = 새 post
    else:
        move_dest = None                     # 자리 잡고 대기, hold

else:
    move_dest = 현재 post
```

## 4. Post 리스트

map1 기준으로 tank 역할별 post를 설정하였다.

| Tank   | Role   | Primary → Alt → Alt            |
| ------ | ------ | ------------------------------ |
| Ally 0 | center | (8, 3) → (0, 30) → (0, -20)    |
| Ally 1 | top    | (28, 18) → (0, 30) → (5, 5)    |
| Ally 2 | bottom | (22, -13) → (0, -25) → (5, -5) |

각 tank는 자기 역할에 따라 primary post로 먼저 이동하고, 이후 조건에 따라 alt post를 순회한다.

---

## 5. Rule bot 세부 규칙

| 항목 | 규칙                                                                                |
| -- | --------------------------------------------------------------------------------- |
| 이동 | 자기 primary post로 이동 → 도착하면 hold → 시야에 적이 없으면 5초 뒤 다음 post 순회 → HP가 낮으면 spawn으로 후퇴 |
| 조준 | 가장 가까운 visible + LOS clear + 아군과 사격선이 겹치지 않는 적을 선택한다. 매 tick 재선택한다.               |
| 발사 | Reload 완료 + aim_target 있음 → 발사한다. 실제 명중 여부는 turret 각도 오차 ±5° 조건에 의존한다.            |

즉 Rule bot은 **“자기 자리로 이동해서 대기하고, 조건에 맞는 가장 가까운 적을 쏘는 방식”** 으로 동작한다.

## 6. Rule bot vs Random Bot 검증

rule_bot.mp4

Random bot은 랜덤 파라미터 모델을 재현하기 위한 bot으로, 무작위에 가까운 행동을 수행한다.

Rule bot과 Random bot을 3 vs 3으로 대결시켜 Rule bot의 기본 성능을 검증하였다.

### 10판 매치 결과

| 지표              | 값             |
| --------------- | ------------- |
| Rule 승률         | 100%          |
| Rule hit rate   | 35%           |
| Random hit rate | 12%           |
| 평균 경기 시간        | 60s, T_MAX 도달 |

10판 실험 결과 Rule bot이 Random bot을 안정적으로 이겼다.

이를 통해 Rule bot이 최소한의 전투 전략과 조준 성능을 갖고 있음을 확인하였다.

## 7. 실제 학습 설정

실제 강화학습에서는 한쪽 team은 고정된 rule bot으로 두고, 반대쪽 team에 역할별 MLP policy를 부여하여 학습한다.

초기에는 rule bot을 imitation learning, 즉 BC로 따라 하게 하여 policy가 완전히 랜덤한 상태에서 시작하지 않도록 한다.

이후 강화학습에서는 다음과 같은 방식으로 self-play 형태의 curriculum을 구성한다.

```text
phase 1:
고정 rule bot 3 vs 학습 policy 3

↓ policy가 rule bot보다 우세해짐

v1 policy 고정

phase 2:
v1 policy 3 vs 새 학습 policy 3

↓ 새 policy가 v1보다 우세해짐

v2 policy 고정

phase 3:
v2 policy 3 vs 새 학습 policy 3

...
```

이런 방식으로 점점 강한 opponent를 만들어가며 policy를 발전시킨다.

## 9. 현재 진행 상황

현재 학습은 다음 설정으로 진행 중이다.

* 한쪽 team: 고정 rule bot
* 반대쪽 team: 역할별 MLP policy
* map: 매 episode마다 10개 map 중 random 선택
* 병렬 환경 수: n_envs 500 ~ 1000
* 학습 방식: BC 이후 강화학습 phase 진행

현재 목표는 rule bot을 안정적으로 이기는 v1 policy를 얻는 것이다.

v1 policy가 확보되면 이를 고정 opponent로 사용하고, 이후 v2, v3 policy를 순차적으로 학습시킬 예정이다.

## 10. 요약

본 파이프라인은 3v3 탱크 전투 환경에서 rule bot 기반 imitation learning과 self-play 강화학습을 결합하는 방식이다.

초기에는 rule bot을 통해 기본적인 이동, 대기, 조준, 발사 전략을 정의하고, 이를 BC로 학습시켜 policy의 초기 성능을 확보한다.

이후에는 고정된 rule bot 또는 이전 버전 policy를 opponent로 사용하여, 새로운 policy가 점진적으로 더 강한 전투 전략을 학습하도록 한다.

또한 하나의 map에 과적합되는 것을 방지하기 위해 기준 map을 jitter하여 여러 variation map을 만들고, 매 episode마다 random map에서 학습을 진행한다.
